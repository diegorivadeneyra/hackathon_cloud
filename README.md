## 2. Reporte de Incidentes

El sistema permite a los usuarios crear reportes de incidentes proporcionando información esencial como el tipo de incidente, la ubicación, una descripción detallada y el nivel de urgencia. Cada vez que se genera un reporte, el backend asigna automáticamente un identificador único utilizando una marca de tiempo en formato ISO, garantizando unicidad y trazabilidad. Los incidentes creados se almacenan en una base de datos serverless DynamoDB, lo que permite un manejo eficiente, altamente disponible y escalable de los registros. Además, cada incidente queda registrado con un estado inicial pendiente, junto con la fecha de creación, el usuario que lo reportó y un historial inicial de acciones que facilita su seguimiento a lo largo del ciclo de vida.
